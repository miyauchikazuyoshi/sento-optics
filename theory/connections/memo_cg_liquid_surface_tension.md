## 核心の観察

CGの液体シミュレーション（SPH, Level Set, FLIP等）は、表面張力を物理的厳密性なしに処理しているが、人間の直感に合うヒューリスティックで構成されている。それが物理と繋がっていても不思議ではない。

実際、CGの処理構造は δ_nuc の言語で自然に再記述できる。

---

## CG表面張力の処理構造

```
CGの標準パイプライン:
  1. 表面を検出（密度閾値 or Level Set φ=0）
  2. 曲率κを計算（∇·(∇φ/|∇φ|)）
  3. γκn̂ の力を Navier-Stokes に注入

暗黙の仮定:
  - γ は定数（外部パラメータ）
  - 表面は「存在する」ことが前提
  - 表面修復は即座（dt内で完了）
```

## δ_nuc による再記述

| CGの処理 | δ_nuc の言語 |
|----------|-------------|
| γ = 定数（ユーザー入力） | δ_elec 由来の凝集力を外部から与えている |
| 粒子が動ける | δ_nuc > 0 が暗黙の前提 |
| 表面修復 | δ_nuc が適切な値 → 拡散で穴を埋める |
| Level Set の φ=0 面 | δ_nuc の空間勾配が急な等値面 |
| 表面張力が「効く」条件 | δ_nuc(dt) ~ O(1)（1ステップ内に隣接距離動ける）|

## δ_nuc の値域と表面の運命

```
δ_nuc 小さい → 修復間に合う → γ維持 → 液体表面
δ_nuc 大きすぎ → 修復追いつかず → 膜破裂
δ_nuc → ∞ → 気体 → 表面消滅
```

修復時間: τ ~ L²/D
δ_nuc(τ) = 6Dτ/a²

これが 1 を超えるか否か = 表面が維持されるか否か。

## CGが暗黙に分離している2変数

CGは δ_nuc と δ_elec を**無意識に分離処理**している：

- **δ_nuc** → 粒子の運動方程式が担保（暗黙的）
- **δ_elec** → γ定数として外部入力（明示的）

2変数を分離して別々に処理しているが、それを意識していない。

## CGで未解決の問題と δ の対応

| CG未解決問題 | δ 言語での解釈 |
|-------------|--------------|
| 薄膜破裂のタイミング | δ_nuc が大きくなりすぎて修復不能になる瞬間 |
| 接触線（3相境界） | δ_nuc(固体)=0 → δ_nuc(液体)>0 の境界条件 |
| 相変化（蒸発/凝縮） | δ_elec → 0 への転移 |
| Marangoni効果 | δ_elec の空間勾配 → 接線力 |
| サブグリッド表面張力 | δ_nuc の局所値で修復能を推定 |

## 実用的な展開可能性

δ_nuc ベースのサブグリッドモデルが作れれば：

1. **薄膜破裂予測**: δ_nuc(t) > 閾値 → 破裂を物理ベースでトリガー
2. **接触線**: δ_nuc の連続変化として自然に記述（スリップ長問題の解消）
3. **相変化**: δ_elec の連続低下として蒸発を記述

現在のCGはこれら全てアドホックなパラメータか経験則。物理ベースの代替ができる可能性。

## 哲学的な示唆

CGのヒューリスティックは「人間の目に自然に見える」ことを基準に作られている。それが物理的に正しい構造と一致するのは、人間の視覚的直感が物理法則の表出を捉えているからかもしれない。

銭湯で「液体が光る」ことに気づくのも同じ構造 — 視覚的直感が δ_elec の物理を捉えている。
